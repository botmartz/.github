We're an AI engineering company. We build two things well: systems that pull reliable answers out of large, messy enterprise document sets (**Enterprise RAG**), and systems that take multi-step action on someone's behalf (**AI Agents**).

Everything else — parsing, chunking, retrieval, evaluation, orchestration, deployment — is *how* we build those two things, not a separate service.

> If a system can't be evaluated in production, we don't consider it shipped.

## What we build

**Enterprise RAG** — Document AI, knowledge search, and internal question-answering over inconsistent, real-world document sets. We treat chunking, retrieval strategy, and evaluation as first-class engineering problems, not a vector database with a prompt on top.

**AI Agents** — Agents with memory across a task, real tool use, planning that adapts when a step fails, and evaluation that tells you whether the task was actually completed. Built for production, not for the demo.

## What we're building here

Public, documented, and reusable. Landing in this org over the coming months:

- **`enterprise-rag-starter`** — a production-shaped RAG baseline, not a toy notebook
- **`langgraph-agent-framework`** — patterns for agent memory, tool use, and planning
- **`fastapi-ai-template`** — a serving layer for RAG and agent workloads
- **`rag-evaluation`** — because "it works in the demo" isn't a metric
- **PyTorch utilities & computer-vision templates**

⭐ Watch this org to see them as they ship.
